# Content Policy and Moderation Rules

**Effective Date:** June 26, 2026

This Content Policy describes the moderation workflows, posting criteria, formatting restrictions, and lifecycle management rules for all content uploaded to **The Helping Society** (the "Platform").

---

## 1. Academic Resources Module

The Resource Hub hosts educational materials to aid the UECU community. All uploads must follow these validation and approval guidelines:

### A. Allowed Content & Constraints
* **Resource Types:** Notes, Past Year Question Papers (PYQs), Books, Syllabus, and general Study Material.
* **Allowed File Types:** PDF, DOCX, PPT, and PPTX documents.
* **Size Limits:** Files must conform to platform-specified file type and size limits.
* **Storage:** Files are stored securely with cloud storage providers and served via secure URLs.

### B. Approval Workflow
* **Student & Core Team Uploads:** Files uploaded by Students enter the **Resource Request Queue** as "Pending."
  * Platform Administrators must review each request in the Admin Dashboard.
  * Admins approve or reject the request. Only approved requests are transferred to the active Resource library.
* **Faculty & Admin Uploads:** Faculty members and Platform Administrators can upload resources directly. These bypass the request queue and go live instantly.

### C. Permissions & DRM (Digital Rights Management)
* **Access Restrictiveness:** Only registered Students, Core Team Members, Faculty, Alumni, and Admins can access study resources. Contributors (recruiters) and Guests (unregistered visitors) are completely blocked.
* **Prohibition of Downloads:** Files must be viewed exclusively within the Platform’s integrated document previewer. File download functionality is disabled.
* **Modification & Deletion:** Only the original uploader or a Platform Administrator can edit or delete a published resource.

## 2. Community Modules

The community board lets users publish peer-to-peer listings. Each sub-module has specific validation rules:

### A. Lost & Found
* **Eligibility:** Only Students and Core Team members can create listings.
* **Requirements:** Title, description, contact number, location, and optional images.
* **Resolution and Auto-Deletion:** When an item is found or returned, the owner marks the post as "Resolved." The system automatically deletes the post after a short temporary timeline once resolved.

### B. Accommodation Room Listings
* **Eligibility:** Only Students and Core Team members can create room posts.
* **Requirements:** Title, description, rental price, location, contact number, and optional images.
* **Auto-Expiry:** Room listings are automatically deleted by the system after a short temporary timeline from creation to keep listings current.

### C. Marketplace
* **Eligibility:** Student, Alumni, and Core Team members can list items for sale.
* **Constraints:**
  * **Price:** Must be non-negative.
  * **Contact Number:** Mandatory.
  * **Images:** Must contain a limited number of images uploaded to cloud storage.
* **Transaction Disclaimer:** All sales occur off-platform. The Platform performs no transaction processing or payment escrow.

## 3. Opportunities Module

Contributors (recruiters, companies) and Admins use this module to share career postings.
* **Live Immediately:** Postings go live immediately upon submission. No pre-approval is required.
* **Allowed Types:** Internships, Jobs, Workshops, Mentorships, and Hackathons.
* **External Links:** Postings must include a valid application link (e.g., job portal, external site).
* **Post-Moderation:** Admins review active opportunities and can delete listings that violate our Community Guidelines.

---

## 4. Comments and Discussion Thread Rules

We enforce plain-text communication to maintain readability and eliminate security vulnerabilities:
* **Format Restriction:** Comments and replies are plain text only. HTML tags, Markdown, or Rich Text styling are prohibited. Any text containing code-injected tags or HTML will be rejected by our database validator.
* **Nesting Depth Limit:** Threads are restricted to a limited nested depth to keep discussions focused and organized.
* **Length Limit:** Comments must conform to platform-specified length limits.
* **Cascading Deletion:** If a parent post (Marketplace, Room, Lost & Found, Resource) is deleted, all comments and nested replies under that post are permanently deleted.

---

## 5. Support and Emergency Requests

The Support Module handles student queries and critical requests:
* **Standard Support Requests:** These are designed for general academic and platform queries. They enter a queue as "Pending" and must be verified by admins. Their status flows from `Pending` -> `Approved` / `Rejected` -> `Resolved`.
* **Emergency Support Requests:** Requests flagged as emergencies bypass the pending queue and go live instantly via an immediate activation workflow.
* **Technical Constraints:**
  * Descriptions are limited to platform-designated character lengths.
  * Location and Contact Number must be provided for emergencies.
  * Users can submit requests anonymously.
